Data-at-Rest Encryption

According to the U.S. Federal Register at 74 FR 42740 (August 24, 2009), data-at-rest "includes data that resides in databases, file systems, flash drives, memory, and any other structured storage method"; and this is in contrast to data-in-motion, data-in-use and data-disposed. Data-at-rest can include HIPPA protected health information, credit card PCI DSS data, or other types of sensitive electronic information that we do not want unauthorized users to have access to. An unauthorized user of stored data could steal or corrupt the breached data, potentially leading to identity theft and other negative consequences of data loss. In order to protect our sensitive data-at-rest from would-be hackers and their prying eyes, it needs to be encrypted.

3 Types of Data-at-Rest Encryption:

  • Full-disk encryption - a password prevents unauthorized access to a device (e.g., after a system boot or when "waking" a sleeping device).
  • File system encryption - a password protects only a particular partition, file system, or even a single directory.
  • Database encryption - a password protects a specific database application, like Oracle or SQL.
    • (Arzt & Berry, 2016)

Microsoft BitLocker Provides Data-at-Rest Encryption

There are numerous products that encrypt data-at-rest. One such product is Microsoft's BitLocker (Vigo, 2018), which makes full-disk encryption as well as USB thumb drive protection possible. Depending on how it is configured, it will either prompt users for a password or it can display an error message that tells the would-be user that the drive cannot be accessed. Unfortunately, it is possible for hackers to use phishing and other social engineering schemes to learn a user's password; so, even strongly encrypted sensitive persistent data can be at risk.
