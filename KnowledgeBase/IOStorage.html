I/O Storage Devices: Hard Drives to Multi-Cloud

I/O Storage Devices Input devices deliver data to a CPU, output devices retrieve data from it, and an input/output device (a.k.a., an I/O device) is a piece of hardware that does both; in particular, a storage device is a type of I/O device (Techopedia, n.d.). The image that I have created below depicts a typical computer framework, consisting of input (keyboard and mouse), output (monitor and printer) and I/O storage devices.

The Hard Disk Drive

Traditional "examples of I/O storage devices are CD/DVD-ROM drives,USB flash drives and hard disk drives" (Techopedia, n.d.). Infact, "the hard disk drive […] has been the main form of persistent data storage in computer systems for decades" (Arpaci-Dusseau & Arpaci-Dusseau, 2012), holding your computer's operating system, applications, and files even when the computer is powered down (TheTeacher.Info, n.d.). However, there are new approaches to input/output storage.

Cloud Storage

An advancement in the persistent I/O storage device model gaining importance is cloud storage. This is where data is uploaded to and accessed from another computer (often a data center belonging to an internet service provider) over a network connection (TheTeacher.Info, n.d.). Google Drive, Dropbox, and Apple iCloud are all examples of free cloud storage. For example, by using Google Drive to generate my discussion post, I am able to access and work on my post from any device, anywhere and know that my work will be backed up and virus protected.

Multi-Cloud

A further advancement in I/O storage, according to Rouse (2017) is the multi-cloud strategy; that is, using more than one cloud provider for infrastructure needs. In fact almost 90% of enterprises now have a multi-cloud strategy for I/O storage (Rayome, 2018) in order to find the right combination of features and services (Asay, 2017). Below, I have further modified the image above to show this newer strategy for persistent storage.

Multi-cloud Integration

An issue with using multi-cloud storage as an I/O device is that of integration, which leads to further advancements. Companies like VMWare, Cisco, and Red Hat are working on a unified and efficient "application layer to span multiple clouds" (Asay, 2017; Forrest, 2018; Rayome, 2018). However, integrating multi-cloud I/O is not a trivial task.
